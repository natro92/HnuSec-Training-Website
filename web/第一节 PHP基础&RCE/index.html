    了解linux

    Linux 是一种开源的操作系统。它就像电脑的“灵魂”，控制着硬件并运行各种软件。更准确地说，Linux 最初指的是操作系统的内核（Kernel），是核心部分。但我们常说的“Linux”，通常指的是基于 Linux 内核构建的完整操作系统，它包含各种工具、应用程序和图形界面等

    根目录/： 它是整个文件系统的顶层目录，所有其他目录都挂载在这个目录下。
    目录（Directory）： 类似于 Windows 的文件夹，用于组织和存储文件。
    文件（File）： 存储实际数据的地方，可以是文本文件、程序文件、图像文件等等。
    路径（Path）： 用于定位文件或目录的字符串，例如 /home/user/documents/file.txt。

    • 绝对路径： 从根目录 / 开始的完整路径，例如 /etc/passwd。
    • 相对路径： 相对于当前目录的路径，例如 documents/file.txt（如果当前目录是 /home/user）

    基础命令

    :::info

    1. ls：列出目录内容

    2. ls：列出当前目录的文件和文件夹。

    3. ls -l：以详细列表形式显示。

    4. ls -a：显示所有文件，包括隐藏文件。

    5. cd：切换目录

    6. cd 目录名：进入指定目录。

    7. cd ..：返回上一级目录。

    8. cd ~：回到用户的主目录。

    9. pwd：显示当前所在目录

    10. mkdir：创建目录

    11. mkdir 目录名：创建一个新的目录。

    12. touch：创建空文件

    13. touch file.txt：创建一个名为 file.txt 的空文件。

    14. cat：查看文件内容

    15. cat file.txt：输出文件内容。

    16. cat flag.txt：常用于查看 flag 内容。

    17. echo：输出字符串或写入文件

    18. echo hello：输出 hello。

    19. echo hello > a.txt：写入到文件（覆盖）。

    20. echo hello >> a.txt：追加到文件末尾。

    21. cp：复制文件或目录

    22. cp a.txt b.txt：复制文件。

    23. cp -r dir1 dir2：递归复制目录。

    24. mv：移动或重命名

    25. mv a.txt b.txt：重命名或移动文件。

    26. rm：删除文件或目录

    27. rm a.txt：删除文件。

    28. rm -r dir：删除目录及其内容。

    29. chmod：修改权限

    30. chmod +x script.sh：给文件添加可执行权限。

    31. ps：查看进程

    32. ps aux：查看所有进程。

    33. ps -ef | grep nginx：查找特定进程。

    34. top / htop：查看系统运行状况

    35. top：实时显示进程状态。

    36. htop：更好用的 top（需安装）。

    37. netstat / ss：查看网络连接

    38. netstat -anp：查看监听端口与进程（老旧系统）。

    39. ss -tunlp：查看网络服务（更现代、推荐）。

    40. ifconfig / ip a：查看网络信息

    41. ifconfig：旧命令，显示 IP 信息。

    42. ip a：现代命令，等效于 ifconfig。

    43. wget / curl：下载文件

    44. wget http://example.com/file

    45. curl http://example.com/file -o file

    46. uname：查看系统信息

    47. uname -a：显示内核和架构等信息。

    48. whoami：查看当前用户名

    49. find：查找文件

    50. find / -name flag.txt：全盘搜索 flag 文件。

    51. grep：搜索字符串

    52. grep root /etc/passwd：搜索包含 root 的行。

    53. export：设置环境变量

    54. export PATH=$PATH:/new/path

    55. which / whereis：查命令路径

    56. which python：查看可执行路径。

    RCE

    RCE（Remote Code Execution，远程命令执行）是指攻击者能够在目标服务器上执行任意代码的一种高危安全漏洞

    刚才我们也学过linux的一些基础命令了，基础命令都是在终端运行的，RCE就是拿到终端的权限，这个时候我们就能对对方的电脑进行控制

    今天主要涉及的内容就是PHP的RCE

    system()：能将字符串作为OS命令执行，且返回命令执行结果；

exec()：能将字符串作为OS命令执行，但是只返回执行结果的最后一行(约等于无回显)

proc_open()：与popen()类似

pcntl_exec()：在当前进程空间执行指定程序；

反引号``：反引号``内的字符串会被解析为OS命令；

assert() 会将字符串当做 PHP 代码来执行；

eval() 会将字符串当做 PHP 代码来执行；

    过滤空格 → 使用 ${IFS}、${IFS%??}、$'\x20'
在 Bash、sh 等 Shell 环境中，IFS 定义了当 shell 解析字符串时用来分隔字段的字符。默认值是一个空格（space）、制表符（tab）和换行符（newline）。
● >  <  <> 重定向符
● %20(space)
● %09(相当于tab键)
● $IFS$9 
● ${IFS}（最好用这个）
● $IFS  
● {cat,flag.txt} 在大括号中逗号可起分隔作用


过滤字母 → 用环境变量组合，比如 ${PATH:0:1}、${HOME:0:1}

URL 编码：%0a, %0d

Base64 编码执行：echo bHM= | base64 -d | bash
